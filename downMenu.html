栈的开发

一:栈的介绍

栈一种先进后出的数据结构（FILO），Java对栈进行了封装。栈的封装类为Stack，继承了Vector类，具体情况如图所示：

Stack类只有一个构造方法。

二:栈的API接口

栈的实例化：Stack <E> stack =new Stack<E>();

empty()与isempty()两个方法是一样的，返回值为boolean类型。当栈为空时，empty方法的返回值为true！另外isempty()是一个继承的方法

int  search(object)

search方法用来搜索参数是否在栈中，参数为Object,可以是任意类型的值。该方法的返回值有两种情况：

  1. 当栈存在该参数时，返回第一个参数在栈中的位置,从上到下，依次为1,2,3...
  2. 当栈不存在该参数时，返回值是-1

E  push(E)

push方法用来向添加栈的元素，返回值为添加的元素

E  pop()

pop方法用来向删除栈顶元素，返回值为删除的元素

E  peek()

peek方法用来查看栈顶元素，返回值为栈顶元素

Git 指令

仓库初始化

  • 设置用户名：git config --global user.name "xmsp1"
  • 设置用户的邮箱：git config --global user.email "1987324004@qq.com"
  • 创建秘钥：ssh-keygen -t rsa -C "1987324004@qq.com"

本地仓库的基本操作

  • 本地仓库的建立：git init
  • 仓库名：.git文件所在目录
  • 增加文件或者修改文件,将文件的修改内容添加到暂存区中：git add test.txt
  • 将暂存区中的所有内容添加到当前分支：git commit -m "添加到当前分区上"
  • 查看当前状态：git status
  • 查看版本：git log
  • 跳转上一个版本： git reset --hard HEAD^
  • 根据ID号，跳转其他版本:git reset --hard 12334455456
  • 查看你的每一条指令：git reflog
  • 撤销修改，在文件修改后，还没存放在暂存区或者文件放在暂存区后，又做了修改的情况：
    git checkout -- test.txt
  • 撤销修改，在文件修改后，存放在暂存区:
    • git reset HEAD test.txt
    • git checkout test.txt
  • 删除文件：rm "test.txt"
    • git rm test.txt
    • git commit -m test.txt
  • 恢复误删的文件：git checkout -- test.txt

分支

  • 创建分支并且切换分支dev：git switch -c dev
  • 切换分支master：git switch master
  • 查看当前分支：git branch
  • 合并分支，要先转到合并的分支上，如master主分支，dev分支：git merge dev
  • 合并完成后，删除分支dev：git branch -d dev
  • 强制删除分支dev：git branch -D dev


  • 将分支dev上传到Github仓库上: git push origin dev

Github远程仓库的操作

  • 上传到Github的两步
    • 绑定远程仓库：git remote add origin git@github.com:xmsp1/.git
    • 上传：git push -u origin master
  • 从Github上下载：git clone git@github.com:xmsp1/.git或者git clone https://github.com/xmsp1/.git

Git服务器

  • 下载：git clone git@47.94.97.145:./test.git
  • 上传：git push origin master
